# Zettelkasten

The Zettelkasten is your knowledge storage.

This is where ideas, concepts and things you learned live.

Unlike the Inbox, this is not a temporary place.
Notes here should be worth keeping.

## The idea

HeadCache uses the idea of small, connected notes.

You don't need to understand the full Zettelkasten methodology to benefit from it, but reading about it is definitely worth it.

The basic idea:

**One note, one idea.**

A note should explain one specific thought or concept in your own words.

## Write for your future self

Do not write notes like you are collecting information.

Write them like you are explaining something to yourself months or years later.

A good note answers:

- What is this about?
- Why is it interesting?
- How does it connect to other things I know?

## Connections over categories

Avoid creating large category trees.

Knowledge is not a folder structure.

Instead of asking:

"Where does this note belong?"

Ask:

"Does this connect to something I already know?"

Link related notes together.

Examples:

- `[[Docker volumes]]`
- `[[Linux namespaces]]`
- `[[Container security]]`

Connections create context.

## Naming notes

Use names that help you find things again.

Prefer clear names over perfect names.

Examples:

Good:
- `docker volumes explain data persistence`
- `ssh keys are authentication credentials`

Bad:
- `Computer stuff`
- `Interesting article`
- `Things about Linux`

Remember:

Search beats hierarchy.

## Sources

Store sources when useful.

A source is not the knowledge itself.

Your note should contain your understanding of the topic.

Example:

```md
## Sources

- Article about Docker volumes
- Documentation about bind mounts

```

## When should something become a Zettelkasten note?

Move something here when:

You learned something worth remembering.
You solved a problem and might need the solution again.
You discovered a useful concept.
You want to connect ideas together.

Do not move everything here.

The Zettelkasten is not a storage place for everything.

Keep only what provides value.

**Remember:**

The goal is not to build the biggest knowledge database.

The goal is to build a useful network of ideas.

Capture.
Connect.
Understand.
Reuse.
